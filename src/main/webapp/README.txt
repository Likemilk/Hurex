사장님~ 읽고 사용해주세요~

/usr/local/develop/tomcat7/webapps/ROOT/resources/property/email어쩌구.txt 파일이 있습니다.

기능 1 : 
	title 은 메일에 보내질 제목을 무엇으로 할지 입니다
	{0} 이것은 컴퓨터에서 읽어서 대체할 문자를 집어 넣는 장소이므로 절대 없어지면 안됩니다.
기능 2 : 
	from 은 사장님께서 직원들에게 메일을 보낼 계정을 말합니다 이때 이 계정은
	어떤 메일 회사든지 메일계정속으로 들어가면 POP3/SMTP 사용 권한에 대한 권한을 관리할 수 있는데
	검색하면 이것에 대한 권한을 설정할 수 있습니다. 여기서 이것을 사용하게 해주어야합니다.
기능 3 : 
	to는 어떠한 것도 필요없습니다 단지 받을 직원들의 이메일 주소를 입력하시면 됩니다.
기능 4 : 
	content는  html 문서형식으로 메일을 보낼 수 있습니다. {0} / {1} / {2} 
	이 내용은 무조건 들어가 있어야합니다
	{0}  = 신청자 이름
	{1}	 = 신청자 전화번호
	{2}  = 신청 날짜 

이 4개의 설정파일들은 서비스가 진행중일때 수정하셔도 별 문제 없게 구현되어있으므로 언제든지 수정하시면 됩니다.	
-----------------------!!NOTICE!!-------------------
 
@이 웹페이지의 조작은  from 에다가 메일을 보내는 용으로 사용할 이메일 계정을 놓습니다. 
그리고 to 에다가는 직원들의 메일주소를 씁니다.

@이메일회사는 gmail이나 naver 꼭 둘중에 하나로 적어주셔야해요~!
만약 다른이메일 회사를 원하신다면 개별적으로 연락해주시기 바랍니다.

@두개 이상 사용할경우 이상한 에러가 발생할거에요 그러므로 꼭 from 에다가는 하나만 사용해주시기 바랍니다~
보내는 용으로 사용할것이기때문에 from 에다가는 하나만 놔서 사용해주시면 감사하겠습니다~

@웹페이지가 실행되고있는 도중에 수정해도 잘 적용됩니다.

@나중에 수정하시려거든 꼭 index.html 의 소스코드를 보고 참고해주시길 바랍니다.
	소스 수정은 ../webapp/WEB-INF/views/index.html 로 넣고 
	이미지같은것들은 전부 ../webapp/resources 에다가 넣어주세요 꼭 이런 사항을 준수 해주어야합니다.
